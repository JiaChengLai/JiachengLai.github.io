MySQL(1)基本语句、数据检索、检索数据排序

Posted on 2018-07-03 | In MySQL

基本语句

显示所有数据库

SHOW DATABASES

连接数据库

USE database_name

显示已连接数据库中所有表

SHOW TABLES

显示数据库中表的结构

SHOW COLUMNS FROM table_name
或
DESC table_name
或
DESCRIBLE table_name
MySQL支持用DESCRIBLE作为SHOW COLUMNS的简写

显示广泛的服务器状态信息

SHOW STATUS

显示创建特定数据库或表的MySQL语句

SHOW CREATE DATABASE database_name
或
SHOW CREATE TABLE table_name

显示授权用户(所有用户或特定用户)的安全权限

SHOW GRANTS

显示服务器错误/警告信息

SHOW ERRORS/SHOW WARNINGS

注意事项

1、SQL语句和大小写 请注意，SQL语句不区分大小写，因此SELECT与select是相同的。同样，写成Select也没有关系。许多SQL开发人员喜欢对所有SQL关键字使用大写，而对所有列和表名使用小写，这样做使代码更易于阅读和调试。

2、使用空格在处理SQL语句时，其中所有空格都被忽略。SQL语句可以在一行上给出，也可以分成许多行。多数SQL开发人员认为将SQL语句分成多行更容易阅读和调试。

数据检索

介绍如何使用SELECT语句从表中检索一个或多个数据列。

检索单个列

检索多个列

检索所有列

等同于

列出所有列名和*号通配符之间的选择
虽然两条语句达到的效果是一样的，但一般不推荐使用*号，而是把列名全部都写出来，在第一条语句执行时，第一条语句会转换成第二条语句再查询数据，增加了查询成本。

使用*号通配符
一般，除非你确实需要表中的每个列，否则最好别使用*通配符。虽然使用通配符可能会使你自己省事，不用明确列出所需列，但检索不需要的列通常会降低检索和应用程序的性能。

根据列名消除重复行

SELECT DISTINCT column_name FROM table_name
DISTINCT必须直接放在列名之前

限制结果

第一行的起始索引为0
检索出来的第一行为行0而不是行1。因此，LIMIT 1, 1将检索出第二行而不是第一行。

在行数问题
在行数不够时,LIMIT中指定要检索的行数为检索的最大行数。如果没有足够的行（例如，给出LIMIT 10, 5，但只有13行），MySQL将只返回它能返回的那么多行。

使用完全限定的表名

SELECT table_name.column_name FROM database_name.table_name

注意事项

SQL语句和大小写
请注意，SQL语句不区分大小写，因此SELECT与select是相同的。同样，写成Select也没有关系。许多SQL开发人员喜欢对所有SQL关键字使用大写，而对所有列和表名使用小写，这样做使代码更易于阅读和调试。
不过，一定要认识到虽然SQL是不区分大小写的，但有些标识符（如数据库名、表名、列名）可能不同：在MySQL 4.1及之前的版本中，这些标识符默认是区分大小写的；在MySQL 4.1.1版本中，这些标识符默认是不区分大小写的。
最佳方式是按照大小写的惯例，且使用时保持一致。

使用空格
在处理SQL语句时，其中所有空格都被忽略。SQL语句可以在一行上给出，也可以分成许多行。多数SQL开发人员认为将SQL语句分成多行更容易阅读和调试。

检索数据排序

以某列的字母顺序排序

SELECT * FROM table_name ORDER BY column_name
例：按商品价格升序排序

按多个列顺序排序

例：按商品价格和商品名称升序排序

指定排序方向

ASC的含义
ASC是ascending(升序),默认排序方式
是升序排序（从小到大,从低到高排序）
DESC的含义
DESC是descending(降序)
降序排序（从大到小,从高到低排序）

例：按商品价格降序排序和商品名称升序排序

ORDER BY和LIMIT结合

使用ORDER BY和LIMIT的组合，能够找出一个列中最高或最低的值。
例：找出products表中价格最高的商品

注意事项

大小写字母排序问题
在对文本性的数据进行排序时,A与a相同吗？a位于B之前还是位于Z之后？这些问题不是理论问题，其答案取决于数据库如何设置。
在字典（dictionary）排序顺序中，A被视为与a相同，这是MySQL（和大多数数据库管理系统）的默认行为。但是，许多数据库管理员能够在需要时改变这种行为（如果你的数据库包含大量外语字符，可能必须这样做）。
这里，关键的问题是，如果确实需要改变这种排序顺序，用简单的ORDER BY子句做不到。你必须请求数据库管理员的帮助。
