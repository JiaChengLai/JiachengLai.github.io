（一）MySQL基本语句、数据检索、数据过滤、通配符

Posted on 2018-07-03 | In MySQL

基本语句

显示数据库结构

SHOW COLUMNS FROM table_name
或
DESC table_name
或
DESCRIBLE table_name
MySQL支持用DESCRIBLE作为SHOW COLUMNS的简写

显示广泛的服务器状态信息

SHOW STATUS

显示创建特定数据库或表的MySQL语句

SHOW CREATE DATABASE database_name
或
SHOW CREATE TABLE table_name

显示授权用户(所有用户或特定用户)的安全权限

SHOW GRANTS

显示服务器错误/警告信息

SHOW ERRORS/SHOW WARNINGS

注意

1、SQL语句和大小写 请注意，SQL语句不区分大小写，因此SELECT与select是相同的。同样，写成Select也没有关系。许多SQL开发人员喜欢对所有SQL关键字使用大写，而对所有列和表名使用小写，这样做使代码更易于阅读和调试。

2、使用空格在处理SQL语句时，其中所有空格都被忽略。SQL语句可以在一行上给出，也可以分成许多行。多数SQL开发人员认为将SQL语句分成多行更容易阅读和调试。

数据检索

消除重复行

SELECT DISTINCT column_name FROM table_name
DISTINCT必须直接放在列名之前

使用完全限定的表名

SELECT table_name.column_name FROM database_name.table_name

以某列的字母顺序排序

SELECT * FROM table_name ORDER BY column_name

按多个列顺序排序

例:按商品价格和商品名称升序排序

指定排序方向

ORDER BY ASCASC是ascending(升序),默认排序方式，是升序排序（从小到大,从低到高排序）
ORDER BY DESCDESC是descending(降序)降序排序（从大到小,从高到低排序）
例:按商品价格降序排序和商品名称升序排序

注意

在对文本性的数据进行排序时，A与a相同吗？a位于B之前还是位于Z之后？这些问题不是理论问题，其答案取决于数据库如何设置。
在字典（dictionary）排序顺序中，A被视为与a相同，这是MySQL（和大多数数据库管理系统）的默认行为。但是，许多数据库管理员能够在需要时改变这种行为（如果你的数据库包含大量外语字符，可能必须这样做）。
这里，关键的问题是，如果确实需要改变这种排序顺序，用简单的ORDER BY子句做不到。你必须请求数据库管理员的帮助。

数据过滤

操作符说明

不匹配检查

SELECT product_id,product_name FROM products WHERE product_id!=1003
或
SELECT product_id,product_name FROM products WHERE product_id <> 1003

空值检查

SELECT product_id,product_name FROM products WHERE product_name IS NULL

IN操作符

IN是WHERE子句中用来指定要匹配值的清单的关键字，功能与OR相当。

SELECT product_id,product_name FROM products WHERE product_id IN (1002,1003)
等价于
SELECT product_id,product_name FROM products WHERE product_id=1002 OR product_id=1003

那为什么要使用IN操作符呢：
 在使用长的合法选项清单时，IN操作符的语法更清楚且更直观。
 在使用IN时，计算的次序更容易管理（因为使用的操作符更少）。
 IN操作符一般比OR操作符清单执行更快。
 IN的最大优点是可以包含其他SELECT语句，使得能够更动态地建
立WHERE子句。

NOT操作符

WHERE子句中的NOT操作符有且只有一个功能，那就是否定它之后所跟的任何条件

SELECT product_id,product_name FROM products WHERE product_id NOT IN (1002,1003)
等价于
SELECT product_id,product_name FROM products WHERE product_id!=1002 OR product_id!=1003

为什么使用NOT？
对于简单的WHERE子句，使用NOT确实没有什么优势。但在更复杂的子句中，NOT是非常有用的。例如，在与IN操作符联合使用时，NOT使找出与条件列表不匹配的行非常简单。

注意

WHERE子句的位置
在同时使用ORDER BY和WHERE子句时，应该让ORDER BY位于WHERE之后，否则将会产生错误。

通配符

任意字符通配符(%号)

例1：匹配商品名称任何位置包含’可乐’的商品

例2：匹配商品名称以’百事’开头，以’乐’结尾的商品

重要的是,要注意到，除了一个或多个字符外，%还能匹配0个字符。%代表搜索模式中给定位置的0个、1个或多个字符。

注意NULL 虽然似乎%通配符可以匹配任何 ，但有一个例外，即NULL。即使是WHERE product_name LIKE ‘%’也不能匹配用值NULL作为产品名的行。

单字符通配符(下划线_)

下划线（_）。下划线的用途与%一样，但下划线只匹配单个字符而不是多个字符。
SELECT product_id,product_name FROM products WHERE product_name LIKE '百事_乐'

注意

搜索是否区分大小写根据MySQL的配置方式，搜索可以是区分大小写的(默认是不区分大小写的)。如果区分大小写，tom与TOM将不匹配。

正如所见，MySQL的通配符很有用。但这种功能是有代价的：通配符搜索的处理一般要比前面讨论的其他搜索所花时间更长。这里给出一些使用通配符要记住的技巧。
 不要过度使用通配符。如果其他操作符能达到相同的目的，应该
使用其他操作符。
 在确实需要使用通配符时，除非绝对有必要，否则不要把它们用
在搜索模式的开始处。把通配符置于搜索模式的开始处，搜索起
来是最慢的。
 仔细注意通配符的位置。如果放错地方，可能不会返回想要的数据。
